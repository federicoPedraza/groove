# Worktree Detail

## Purpose
Worktree Detail is the focused view for one worktree. Use it when you need terminal actions, runtime controls, task assignment, and branch-specific context.

## Core actions
The detail page surfaces open terminal, run local command, and play Groove command for the selected worktree. Commands use the workspace command templates and substitute the current worktree path.

## Task assignment
A worktree can be linked to a workspace task so planning and execution stay aligned. Assignments are stored in workspace metadata and shown on both dashboard and detail views.

## Branch and sync context
Detail highlights branch, path, and status signals for quick inspection. If the row becomes stale or corrupted, use diagnostics and recovery guidance before deleting.

## Recommended flow
Open detail from dashboard, run local checks, complete the assigned task, then return to dashboard for merge and cleanup workflow.
